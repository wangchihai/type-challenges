> Traduzido pelo Google, abra um PR para ajudar a melhorar a tradução.

Opções encadeáveis ​​são comumente usadas em Javascript. Mas quando mudamos para TypeScript, você consegue digitá-lo corretamente?

Neste desafio, você precisa digitar um objeto ou classe - o que quiser - para fornecer duas funções `option(key, value)` e `get()`. Em `option`, você pode estender o tipo de configuração atual pela chave e valor fornecidos. Devemos acessar o resultado final via `get`.

Por exemplo

```ts
declare const config: Chainable

const result = config
  .option('foo', 123)
  .option('name', 'type-challenges')
  .option('bar', { value: 'Hello World' })
  .get()

// expect the type of result to be:
interface Result {
  foo: number
  name: string
  bar: {
    value: string
  }
}
```

Você não precisa escrever nenhuma lógica js/ts para lidar com o problema - apenas no nível de tipo.

Você pode assumir que `key` aceita apenas `string` e `value` pode ser qualquer coisa - apenas deixe como está. O mesmo `key` não será aprovado duas vezes.
